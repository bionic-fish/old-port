Travel Application

Overview

A mock up for a hotel booking website + application. As a fond traveller and adventurer, it is always a blessing to have a hotel application that can find something to suite ones aquired tastes. In using a variety of hotel booking applications, I decided I would create my own in order to expand my skill set in form creation and user input validation.


Having a booking app that is responsive across multiple devices was a top priority. Some forms behave weird when viewed on a phone or screen smaller than 300px wide. Taking the inititive to build a fully responsive hotel booking form and gain extra knowledge about web based services.



The Problem

Viewing an array of forms on smaller devices can sometimes be a hassel if the form contains a lot of elements. To address this issue commonly found throughout hotel booking apps, I created a responsive form that splits particular sections for easier viewing of the elements.


Another issue is some forms dont always save a users input after submit or a next page with can incur the user to visit and resubmit the same form multiple times. This issue was addressed by attaching form validation to all elements that require a specific piece of information to be passed along to the server for reuse. E.g email and attached user name for login purposes.


Design

By replicating other hotel booking apps and applying my own flair to the final design, I have come up with a sleek and professional looking hotel booking application that can be used when travelling abroad. The dark colored elements really help make the scenic photographs stand out as vibrant pieces of the webpage. By giving a light colored background, this also enhances the richness of the travel photographs in a pleasing way.


The design features a mobile view and a desktop view both with responsive elements up until the screen dimensions of 3440x1440. Cafe menus and forms recieve a background color and additional properties on desktop view.


Development

Created by researching and taking inspiration from other hotel design application and websites to develop a better understanding of development procedures attaining to web based technology. Developed the form validation using a mix of HTML5 input properties and JavaScript.


Taking use of the display flex property in CSS to layout specific content depending on a users viewing device. Mock up created as a personal project to develop greater proficency in programming web technology languages.



Testing / Debugging

Testing takes place using multiple browsers to test compatibility. Lauching on both a local server for development and testing purposes, and creating live sites on github to test features.


Multiple devices are also used for testing to ensure there are specific media query breakpoints in place. This ensures a responsive design and layout across multiple browsers and devices.


Comments

Creating this website helped develop a better understanding of layout principals through the use of CSS Flexbox techniques.
Provided an insight into image placement, design principals for UX / UI elements and form creation and validation. color
